# dismissItemReturnRequestValidationStep - Medusa Core Workflows Reference

This documentation provides a reference to the `dismissItemReturnRequestValidationStep`. It belongs to the `@medusajs/medusa/core-flows` package.

This step validates that a return request can have its items dismissed.
If the order or return is canceled, the order change is not active, 
or the items do not exist in the return, the step will throw an error.

:::note

You can retrieve an order, return, and order change details using [Query](https://docs.medusajs.com/learn/fundamentals/module-links/query),
or [useQueryGraphStep](https://docs.medusajs.com/resources/references/medusa-workflows/steps/useQueryGraphStep).

:::

## Example

```ts
const data = dismissItemReturnRequestValidationStep({
  order: {
    id: "order_123",
    items: [
      {
        id: "orli_123",
        // other item details...
      }
    ]
  },
  orderReturn: {
    id: "return_123",
    // other return details...
  },
  orderChange: {
    id: "orch_123",
    // other order change details...
  },
  items: [
    {
      id: "orli_123",
      quantity: 1,
    }
  ]
})
```
